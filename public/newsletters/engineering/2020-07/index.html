‚ú® Tendermint Core

Lead by Interchain GmbH and Informal Systems


Stargate's upcoming serialization format change (Protobuf) in Tendermint Core 0.34 is a state-breaking change. Tendermint Core users will have to conduct a network upgrade to use Tendermint Core 0.34 or later. If this is new for you, here are two helpful resources to get started:


  • Blog post from Tess Rinearson: ‚ÄúTendermint 0.34, Protocol Buffers, and You.‚Äù 

  • An Tendermint Core upgrade guide to help move your project transition to the newer version.


If you are using Tendermint Core you'll want to keep an eye on:


  • Amino to Protobuf migration for JSON generation: Currently, Tendermint Core uses Amino to generate JSON files. This will impact many applications when Protobuf is used to generate JSON. 

  • ABCI v0.17.0: This does require minor code changes mostly to snapshots. You‚Äôll have to implement these snapshot methods within your application if you want it to state sync. üëâ ADR-053 on the topic

  • Tendermint state sync: The pull request is now merged and new nodes will sync 200x faster than before.

  • Cosmos SDK IBC implementation: The IBC Tendermint client requires more changes in order to work with Tendermint 0.34.

  • Testnets for upcoming release candidates: üëâ Updates on 0.34 testnets

‚ú® CosmWasm

Lead by Confio


The recent 0.9 release included a number of features to help developers start designing and building smart contracts with CosmWasm. 

üëâ Full blog post 



What's included?

Cross-Contract Queries: execute arbitrary read-only queries from your contract to other contracts, and to native Cosmos SDK modules.

Custom Native Modules: You can provide bindings to any of your go-sdk modules and chain-specific contracts will be able to access that interface - both querying as well as sending messages to your contracts.

Contract Migrations: What is called ‚ÄúUpgradable Proxy Contracts" in Ethereum, but more stable and secure.

Storage iterators: This allows more specific data for example instead of ‚ÄúHow many tokens does account X have?‚Äù you could ask ‚ÄúShow me all addresses with a balance over 100 coins‚Äù. 

CosmJS Client Lib: a new TypeScript library to power CosmWasm clients.üëâ Blog post

Demo Net: Stay tuned for more details on how you can connect, hit the faucet, and upload and experiment with your own contracts. 


What‚Äôs next?


For the Stargate upgrade, CosmWasm will have full support for IBC and the Protobuf changes.


The permission system being added for CosmWasm 0.10 is part of the Cosmos Hub Proposal that will let chains optionally control the contract lifecycle with on-chain governance (shutdown/upgrade ‚Äúbad‚Äù contracts or require approval to even deploy contracts). 


The CosmWasm 1.0 release is slated for August. The only new feature is the permissioning, and all other work is being done to provide maximum stability and security to the system. We hope to release this based on topics outlined in Launchpad.
