Hermes 0.6 release

Since the initial 0.1 release, we’ve released 4 more major versions of Hermes: 0.2, 0.3, 0.4, 0.5 and 0.6 Hermes can now:

  • Relay packets concurrently over multiple paths

  • Clear pending packets when (re)starting

  • Expose metrics to Prometheus via its built-in telemetry service

  • Complete channel handshakes

  • Support ordered channels, packet delays, timestamp-based packet timeouts, and Interchain Accounts (ICS 027)

  • Perform misbehaviour detection and evidence submission

  • Perform client upgrades after counterparty chain upgrades

  • Periodically refresh IBC clients in the background

  • Hold multiple signing keys in the file-based keyring

  • Dynamically estimate the gas needed to submit a transaction

  • Detect chain impersonation attacks

  • Configure to be reloaded by sending the Hermes process a SIGHUP signal (#1117)

  • Filter based on client state trust threshold (#1165)

Since Osmosis launched on June 19, Hermes is running between the Osmosis chains and 7+ other networks! The relayer is successfully transferring packets between channels to and from IBC connected chains. We continue to make improvements on Hermes and add functionality to the relayer that will allow it to successfully relay packets on future Cosmos DEXs and AMMs. 

Discover the open issues (including those that are resolved) which we have been working on in preparation for the Gravity DEX launch in early July. Future features and improvements include:


  • Improved relayer intelligence including support for identifying whether it is compatible with chain version

  • Improved UX including a bootstrapped implementation of an architecture for hermes operation management including

Starport

Led by Tendermint Inc

Released Starport v0.15 with IBC scaffolding and TypeScript/Vuex code generation and Starport v0.16. 

The new Starport v0.16 included:

  • protoc is now bundled with Starport CLI. You don’t need to install it anymore.
  • 'starport relayer' 'configure' and 'connect' commands now use the confio/ts-relayer under the hood. Also, checkout the new 'starport relayer lowlevel' command.
  • An OpenAPI spec for your chain now automatically generated with 'serve' and 'build' commands. A console is available at {localhost:1317} and spec at {localhost:1317/static/openapi.yml} by default for the newly scaffolded chains.
  • Keplr extension is supported on web apps created with Starport
  • Tests to the scaffold
  • Ability to scaffold modules in chains not created with Starport
  • Ability to scaffold Cosmos SDK queries
  • IBC relayer support is available on web apps created with Starport
  • docs command to read Starport documentation on the CLI
Other updates from the Starport team:
  • Published documentation
  • Created a monorepo for Starport Interface (SPI) Vue component and Vuex modules
  • Wrote two tutorials: IBC Hello world and Interchain Exchange
  • Participated in events: Blockchain Tech 2021 (in Russian) and Hackatom RU (in Russian)
  • Successfully demoed interchain token transfers live during IBC Launch Party

Our next priorities:

  • Adding more features to Starport Network
  • Improving client-side code generation and Vue components
  • Refactoring Starport CLI codebase
